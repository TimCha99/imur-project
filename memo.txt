[깃허브 사용 시 주의사항]
1. 작업 시작 전 항상 최신상태로 만들고 시작한다.
안그러면 나중에 다른 컴퓨터에서 작업 시 로컬과 원격 저장소의 기록이 엇갈린다.
 >> git pull origin master
pull 명령어는 fetch 와 merge를 동시에 수행하는 명령어

2. 작업 후, 3단계 공식으로 업로드!
 >> git add "작업 파일/폴더"
 >> git commit -m "작업 내용 요약"
 >> git push

3. 사이트에서 수정 후 로컬 작업 전 반드시 git pull 실행
로컬에서 수정 후에는 [2]의 3단계 공식으로 업로드한다.

4. 빈 폴더는 올릴 수 없음
git 은 파일만 추적하기 때문에 비어있는 폴더에는 git 이 인식할 수 있는 파일을
하나 만든다.
 >> touch .gitkeep
